# Swing Options Tracker Setup

## Overview

The `swing-options-tracker` service runs daily at **3:45 PM Eastern** (market days only) and:
1. Analyzes 50+ large/mega-cap stocks for swing trade opportunities
2. Suggests ONE high-probability swing option trade
3. Tracks P/L of previous trades until expiry (5 days)
4. Posts status updates to Discord webhook

## How It Works

### Trade Scoring

Stocks are scored based on:
- **5-day momentum** (% change)
- **Volatility** (intraday fluctuations)

Top-scoring stock gets the daily suggestion.

### Trade Details

Each suggested trade includes:
- **Type**: CALL (bullish) or PUT (bearish)
- **Strike Price**: At-the-money or slightly out-of-money
- **Target**: +5% to +15% profit
- **Stop Loss**: -2% to -5%
- **Expiry**: 5 days from entry

### P/L Tracking

Open trades are updated daily:
- **HOLD**: Trade moving as expected, no target/stop hit
- **SELL - Target reached**: Profit target hit (+5-15%)
- **SELL 50% / HOLD 50%**: Halfway to target, suggests partial profit-taking
- **SELL - Stop loss hit**: Loss hit (-2-5%), exit recommended
- **EXPIRED**: Trade past expiry date (5 days)

## Seasonal Adjustment

**Important**: The cron schedule assumes EDT (UTC-4). During winter months (November-March), EST (UTC-5) applies, so adjust:

```javascript
// In ecosystem.config.js, change to:
cron_restart: "45 20 * * 1-5",  // Winter (EST): same as 3:45 PM EST
```

Then:
```bash
pm2 reload ecosystem.config.js
pm2 save
```

## Troubleshooting

### No Discord messages

1. Check `.env.production` has the webhook URL
2. Test the webhook URL manually:
   ```bash
   curl -X POST -H "Content-Type: application/json" \
     -d '{"content":"Test message","username":"Swing Options Tracker"}' \
     https://discord.com/api/webhooks/.../...
   ```

### Service not running

1. Check PM2 logs:
   ```bash
   pm2 logs swing-options-tracker
   ```

2. Test manually:
   ```bash
   cd /home/ubuntu/Thalaivas-Cricket-Team/stock-analysis-mcp
   npm run swing:options:once
   ```

3. Verify cron is correct (should match current time zone):
   ```bash
   date  # Check server time
   pm2 show swing-options-tracker
   ```

### Trades not updating

Ensure `.swing-options-state.json` exists and is writable:
```bash
ls -la /home/ubuntu/Thalaivas-Cricket-Team/stock-analysis-mcp/.swing-options-state.json
```
